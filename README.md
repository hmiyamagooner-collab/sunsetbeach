# SUNSET BEACH PARK INAGE ・ 公式アプリ（プロトタイプ）

稲毛海浜公園（SUNSET BEACH PARK INAGE）の来園者向けアプリの試作です。
コンセプトは **「並ばない稲毛」** ── 夏のプールで、飲食を事前に予約して、現地では並ばず受け取るだけ。

トップの夕景から「飲食店メニュー」を押すと、予約画面がひらき、予約すると受け取り券（PICKUP PASS）が出て、券からまたトップに戻る。この一続きの流れが、`index.html` ひとつの中で動きます。

---

## 動かし方

`index.html` をブラウザで開くだけです。インストールもサーバーも要りません。スマホに送って開けば、実機の見え方も確認できます。

---

## いま動くこと / まだ動かないこと

- **動く**：トップ画面、飲食の事前予約、受け取り券の発行、本物の園内マップへの遷移
- **まだ動かない**：予約はいまブラウザの中だけに保存されます。**業者さんのスマホにはまだ飛びません。** ここを繋ぐのが次のステップ（下記）です
- マップ以外のボタン（チケット・イベント・施設案内・歩いてためる・ゲーム）は、押すと「これから開発する画面です」と表示されます

---

## カスタマイズできる場所

すべて `index.html` の中にあります。テキストエディタで開いて、以下を書き換えてください。

| 変えたいもの | 場所（`index.html`内） |
|---|---|
| メニューの品名・価格・アイコン | 下の方の `const MENU = [...]` |
| 受け取り時間の選択肢 | `const TIMES = [...]` |
| 色（サンセットのオレンジなど） | 上の方の `:root { --sunset-3: ... }` などの変数 |
| ヒーローの夕景写真 | `.hero .photo` の `background-image`（現在はbase64で埋め込み済み） |
| マップの遷移先 | マップボタンの `onclick="window.open('...')"` |

色は `--sunset-3`（濃いオレンジ／ボタン色）を軸に、公園マップのテーマカラー `#DB531D` と揃えてあります。

---

## 次のステップ（本物化）

1. **予約を業者に飛ばす**：Supabase を入れて、来園者の予約を業者さんのスマホの注文一覧に届くようにする
2. **配れるURLにする**：Vercel にデプロイして、来園者にも業者にもURLで配れるようにする
3. **メニューを本物に**：実際のキッチンカーのメニューと価格に差し替える
4. **夏の限定テスト**：前向きな業者さん1台と、特定の週末だけで試して、行列が消えるか・売上が動くかを数字で見る

> データベースへの接続やデプロイは、一度やると戻しにくい作業です。着手のときは内容を確認してから進めてください。

## GitHub への上げ方（かんたん）

1. GitHub で新しいリポジトリを作る（New repository）
2. 「uploading an existing file」から、`index.html` と `README.md` をドラッグ＆ドロップ
3. Commit する

これで、いつでも開いて育てられる状態になります。以降の変更は、ファイルを差し替えて Commit するだけです。
